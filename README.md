# Cerberus - AI-Powered DeFi Agent

**AgentKit Hackathon 2025 Entry**

Cerberus is an autonomous DeFi intelligence platform that combines AI-powered market scanning, secure biometric verification, encrypted messaging, and on-chain payments to create a new paradigm in trustless trading assistance.

## Why This Architecture?

### Minimal Database Approach
- ✅ **XMTP stores messages** - No duplication needed
- ✅ **On-chain stores payments** - Only ephemeral state in Redis
- ✅ **AI agent computes live** - No stale cached data
- ✅ **Only Convex for**: World ID nullifier tracking (sybil resistance)

### Key Design Decisions
1. **Convex over Supabase** - Serverless functions built-in, no API routes needed
2. **Redis for x402** - Ephemeral payment state with TTL
3. **IndexedDB for XMTP** - Local cache, network is source of truth
4. **AgentKit for AI** - Real-time market scanning without persistence

## Implementation Notes

### XMTP WASM Handling
XMTP requires WASM bindings that don't work during SSR. The implementation uses:
- Dynamic imports (`await import('@xmtp/xmtp-js')`)
- Client-side only execution (`typeof window !== 'undefined'`)
- Turbopack externals configuration

### Convex Gradual Adoption
The codebase supports both:
1. **API routes** - Work immediately without Convex setup
2. **Convex functions** - Replace API routes once initialized

Hooks gracefully fall back to API routes when Convex isn't available.

### x402 Payment State
Payments use Redis with TTL (time-to-live):
- Pending payments expire after 15 minutes
- Settled payments retained for 1 hour (then auto-expire)
- No permanent storage needed - on-chain is source of truth
